Data Cleaning and EDA Plan for Salary Prediction Project
Dataset Overview
The ds_salaries.csv contains 607 records of data science job salaries with the following columns:
- work_year: Year of salary data (2020-2022)
- experience_level: EN (Entry-level), MI (Mid-level), SE (Senior-level), EX (Executive)
- employment_type: FT (Full-time), PT (Part-time), CT (Contract), FL (Freelance)
- job_title: Specific job titles (100+ unique values)
- salary: Salary in local currency
- salary_currency: Currency of salary (USD, EUR, GBP, etc.)
- salary_in_usd: Target variable - salary converted to USD
- employee_residence: Employee's country of residence
- remote_ratio: Percentage of remote work (0, 50, 100)
- company_location: Company's country location
- company_size: S (Small), M (Medium), L (Large)

- Identify missing values (visual inspection suggests clean data, but will verify)

- Categorical columns: Mode imputation or "Unknown" category
- Special handling for job_title if missing (may group as "Other")
3. Data Validation
- Verify salary_in_usd correctness: Should equal salary when salary_currency = 'USD'
- Check for inconsistent currency conversions
- Validate work_year range (2020-2022)
- Confirm remote_ratio values are only 0, 50, 100
- Ensure experience_level values are only EN, MI, SE, EX
4. Feature Engineering Preparation
- Create ordinal mapping for experience_level: EN=0, MI=1, SE=2, EX=3
- Consider grouping rare job_title values (appearing <5 times) into "Other"
- Consider creating geographic regions from country codes
- Note: salary column may be redundant for modeling since we have salary_in_usd
5. Outlier Analysis
- Examine salary_in_usd distribution for extreme values
